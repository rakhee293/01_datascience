F1 - Score:¶

  • F1-Score is a combination of Precision and Recall. F1-Score is a harmonic mean of precision and recall (harmonic mean gives more weight to low value). As a result, the classifier will give high F1-Score only if both Precision and Recall are high. \begin{equation} F1 = \frac{2} {\frac{1} {Precision} + \frac{1} {Recall}} = \frac{TP} {TP + \frac{FN + FP} {2}} \end{equation}
  • F1-Score is a simple way to compare two classifiers.
In [159]:
from sklearn.metrics import f1_score
f1_score(digit_y_train_lbl, pred_y_train)
Out[159]:
0.7593958552862663
  • F1-Score favors classifiers that have similar Precision and Recall. This is not always what we want, in some cases high Precision is prefered, in other cases high Recall is prefered
  • Example "High Precision" prefered case : Healthy Food classifier should have High Precision (Healthy food classification should be correct(High Precision), It is fine to miss-classify some healty food as unhealthy(Low Recall)
  • Example "High Recall" prefered case : Heart Attach probability calssifier should have High Recall (have to classify all future Heart Attack patient correctly(High Recall), may incorrectly predict some as future Heart Attack patients).

Precision-Recall Trade Off : Increasing Precision reduces Recall, and vice versa. This is called as precision-recall trade off¶

  • Scikit-Learn provides a way to take controle over Precision, Recall scores. We can use method=decision_function in cross_val_predict and get decision scores. Try different thresholds (a number) to find different Precision, Recall scores and settle at a satisfying threshold.

  • An easier way to find best threshold for desired Precision or Recall is to plot graph between Precision/Recall VS Threshold (precision_recall_curve).

ROC (Receiver Operating Characteristic) Curve: The plot between TRUE POSITIVE RATE (recall/sensitivity) and FALSE POSITIVE RATE (1 - true negetive rate). (True Negetive Rate is also known as "Specificity")

  • TRUE POSITIVE RATE (TPR or Sensitivity or Recall) = $\frac{TP} {TP + FN} $
  • FALSE POSITIVE RATE (FPR) = $\frac{FP} {FP + TN} $
  • TRUE NEGETIVE RATE (TNR or Specificity) = $\frac{TN} {TN + FP} $
  • This curve is commonly used with binary classifier.

  • ROC curve is so similar to the Precision-Recall curve, you may wonder how to decide which one to use. As a rule of thumb, you should prefer the Precision-Recall curve whenever the positive class is rare (as in the digit recognizer dataset)or when you care more about the false positives than the false negatives, and the ROC curve otherwise.
